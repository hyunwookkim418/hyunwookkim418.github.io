W1. PoE Power Insufficiency Causing AP to Operate in 2.4 GHz–Only Mode

Last updated: 1/29/26

Problem Summary

Several wireless access points were observed operating only on the 2.4 GHz band, with the 5 GHz radio disabled. Initial suspicion pointed to Radio Frequency (RF) configuration or controller settings, but further investigation revealed the root cause to be insufficient PoE power allocation at the access switch port.

The AP’s 5 GHz radio was disabled not by RF or controller settings, but by PoE under-allocation at the switch port.

Why it matters

Modern enterprise APs dynamically disable high-power radios when available PoE power falls below the required threshold.

  • 2.4 GHz → lower power requirement
  • 5 GHz → higher power requirement
  • PoE under-allocation → AP enters degraded mode
Result: Reduced wireless performance and capacity due to loss of 5 GHz service.

Environment

  • Core Switch: Cisco Catalyst 9500 16X 10G (Backbone)
  • Access Switch: Cisco Catalyst 9200L 24 PoE+
  • Access Point Vendor: Hewlett-Packard Aruba AP-535 (802.11ax, 2.4 GHz / 5 GHz)
  • Issue Type: PoE budget / power negotiation
  • Impact: Partial wireless service degradation (5 GHz unavailable)

Lessons Learned

  • PoE is not just power delivery — it directly affects device functionality.
  • RF issues may originate from Layer 1 electrical constraints.
  • Always verify PoE budget and negotiation before changing RF or controller configurations.

Troubleshooting Process

1

Identify the Affected Device (L3 → L2 Trace)

CLI Output 
C9500_BB# show arp | include 192.168.123.123
C9500_BB# show mac address-table address 6026.efc0.78b8
  • Replace 192.168.123.123 with the IP address of the target device you are troubleshooting.
  • Replace the MAC address with the actual device MAC, such as 6026.efc0.78b8.
  • This confirms how the switch learned the endpoint MAC through the ingress interface path.
2

Locate the Access Switch via CDP

CLI Output 
C9500_BB# show cdp neighbors
C9500_BB# show cdp neighbors detail
Determines the connected access switch and its management IP for the next hop in troubleshooting.
3

Access the Edge Switch and Verify PoE Status (Critical Step)

CLI Output 
P1_Incoming_01# show power inline
  • The port showed PoE active but under-allocated.
  • The AP powered on, but it was not receiving sufficient wattage for full functionality.
4

Verify MAC Address on the Access Switch

CLI Output 
P1_Incoming_01# show mac address-table address 6026.efc0.78b8
Confirms the exact switchport where the device is connected, such as Gi1/0/13.
5

Review Interface Configuration

CLI Output 
P1_Incoming_01# show run interface Gi1/0/13
Displays interface-level settings, including PoE behavior, VLAN assignment, speed, and duplex.
6

Resolution: Explicitly Allocate Adequate PoE Power

CLI Output 
P1_Incoming_01# configure terminal
P1_Incoming_01(config)# interface Gi1/0/13
P1_Incoming_01(config-if)# power inline static max 30000
P1_Incoming_01(config-if)# end
P1_Incoming_01# write memory
This sets the PoE budget for the port to 30 watts, ensuring sufficient power for devices such as enterprise access points or IP cameras.
